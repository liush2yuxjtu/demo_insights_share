❯ explain to my teammates how to save .claude/plans to project level            

⏺ 把 .claude/plans/ 落到项目级 — 给队友的说明

  一句话原理

  在仓库根目录创建 .claude/settings.json，写入 plansDirectory 字段，Claude Code
  写 plan 文件时就会落到这个相对路径，而不是默认的全局/临时位置。

  三步落地

  1. 建配置文件 .claude/settings.json：
  {
    "plansDirectory": ".claude/plans"
  }

  2. 提交进 git（关键，这样才能跨电脑共享）：
  git add .claude/settings.json
  git commit -m "chore: 设置 plan mode 落地到项目目录"

  .claude/plans/ 目录会在第一次写 plan 时由 Claude Code 自动创建，无需手动
  mkdir。

  3. （可选）让 plans 也进 git

  两种风格，按团队偏好二选一：

  ┌─────────┬────────────────────────────┬──────────────────────────────────┐
  │  风格   │            做法            │             适用场景             │
  ├─────────┼────────────────────────────┼──────────────────────────────────┤
  │ 共享    │ 不加 ignore，plan          │ 想做架构评审、PR 里引用          │
  │ plan    │ 文件也提交                 │ plan、留痕                       │
  ├─────────┼────────────────────────────┼──────────────────────────────────┤
  │ 本地    │ 在 .gitignore 加           │ 只想个人记录，不污染仓库历史     │
  │ plan    │ .claude/plans/             │                                  │
  └─────────┴────────────────────────────┴──────────────────────────────────┘

  为什么这么配

  - 路径是相对的 → 每个队友 clone 后都自动指向各自仓库内的
  .claude/plans/，无需改配置
  - 跟着仓库走 → 切换电脑、换 worktree、换分支都自动生效
  - 可被 review → plan 是 markdown，PR diff
  直接能看，比"先口头对齐再写代码"留痕更清晰

  验证生效

  让任何一个队友在项目根目录起一次 plan mode 任务，结束后检查：

  ls .claude/plans/

  应该能看到形如 curious-strolling-wave.md 这种带随机名的 plan
  文件。我们刚才那次就是命中这个机制 — 当前仓库就有一份样例：

  .claude/
  ├── settings.json              ← 配置入口
  └── plans/
      └── curious-strolling-wave.md  ← 上一次 plan 留下的样例

  小提醒

  - 配置文件名不要写成 settings.local.json（那是个人覆写，不入 
  git）；要让全队生效就用 settings.json
  - plansDirectory 不支持 ~ 展开，用相对路径最稳
  - 如果队友本地已有 ~/.claude/settings.json 设了别的
  plansDirectory，项目级会覆盖全局级，所以无需提醒他们改全局配置
