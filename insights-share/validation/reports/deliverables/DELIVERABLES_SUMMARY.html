关键发现

WITH vs WITHOUT — 字数减少 35%，信息密度反而上升。

  • WITH 轮直接定位到 alice_pgpool.json 的 2026-04-10 事故，给出 idle_in_transaction_session_timeout = '30s' 这种「按本团队既往经验」的具体止血参数。
  • WITH 轮显式声明 do_not_apply_when: session pooling mode 反向适用边界，避免在错误场景下误用。
  • WITHOUT 轮开场即承认「项目里没有 checkout 或 PostgreSQL 相关代码」，只能罗列 pg_stat_activity 三段查询 + 四种连接池配置示例 + 反问，完全是教科书答案。
  • 结论：insights-wiki skill 一旦在 ~/.claude/skills/ 注册，Claude Code 会在收到匹配主题的问题时自动调用 — demo 的核心价值「局域网集体智慧自动注入」由本对照实验证实。
